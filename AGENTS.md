# AGENTS.md

Este proyecto sirve para crear y vender páginas web sencillas, modernas y profesionales para pequeños negocios locales usando VS Code + Codex.

## Objetivo del proyecto

Vender el servicio:

**Web Express Premium para negocio local**

El servicio consiste en una landing web de una página para negocios pequeños, responsive, rápida, clara, con WhatsApp/contacto, SEO básico, diseño profesional y publicación online.

## Precio

- Precio piloto: 390 € para los 2 primeros clientes.
- Precio estándar después: 590 €.
- Mantenimiento opcional: 49 €/mes.

## Reglas técnicas

- Usar HTML, CSS y JavaScript puro.
- No usar backend.
- No usar base de datos.
- No usar frameworks salvo que se pida expresamente.
- No usar React, Vue, Angular ni Bootstrap.
- No usar n8n.
- No usar scraping.
- No crear dashboards.
- No crear automatizaciones complejas.
- No tocar facturas ni datos sensibles.
- Priorizar webs estáticas fáciles de publicar en GitHub Pages, Netlify o Vercel.

## Reglas de diseño

- Diseño profesional, limpio y moderno.
- Excelente versión móvil.
- Buen uso de espacios.
- Tipografía clara.
- Botones visibles.
- CTA de WhatsApp, llamada y contacto.
- No parecer una plantilla barata.
- No parecer un ejercicio escolar.
- La web debe poder enseñarse a un cliente real.

## Reglas comerciales

- No vender “web con IA”.
- No mencionar Codex ni herramientas internas al cliente.
- Vender claridad, confianza, imagen profesional y facilidad de contacto.
- No prometer clientes garantizados.
- No prometer SEO avanzado si solo se hace SEO básico.
- No regalar cambios infinitos.
- No empezar trabajos sin alcance claro.

## Demo principal

La primera demo será para un centro ficticio de fisioterapia local.

Debe incluir:

- Hero profesional.
- Servicios.
- Método de trabajo.
- Confianza.
- Reseñas ficticias marcadas como demo.
- Ubicación.
- Horarios.
- FAQ.
- CTA WhatsApp.
- CTA llamada.
- SEO básico.
- Schema.org.
- Footer legal básico de demo.

## Web pública y portfolio comercial

La carpeta `web/` contiene la web pública preparada para publicación en Netlify.

Debe incluir el portfolio comercial, las páginas legales y las demos ficticias mostradas dentro del portfolio.

Netlify debe publicar solo `web/`.

La carpeta `tools/` contiene herramientas internas y no debe copiarse dentro de `web/`.

No subir `.env`, `.venv`, datos internos, CSVs privados, bases de datos locales, logs ni credenciales.

Debe explicar:

- Qué vendo.
- Para quién es.
- Qué incluye.
- Qué no incluye.
- Precio piloto.
- Demo de fisioterapia.
- Caso real mluz_fotografia_ como referencia técnica/visual.
- Proceso de trabajo.
- Preguntas frecuentes.
- Contacto.
